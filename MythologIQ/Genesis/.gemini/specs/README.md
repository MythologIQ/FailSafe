# The Specification Registry

This directory contains the "Blueprints" for all significant changes in Genesis v2.

## Protocol

1.  **Never Code Without a Spec**: If a task involves > 50 lines of code or > 1 file, create a spec first.
2.  **Use the Template**: Copy `_TEMPLATE.md` to `[feature-name].md`.
3.  **Get Approval**: The User must approve the Spec before `implementation_plan.md` is created.
4.  **Immutable History**: Once implemented, Specs serve as documentation of "Why" things work the way they do.
