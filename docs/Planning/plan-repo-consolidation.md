# Plan: Repository Consolidation

**Risk Grade**: L2
**Author**: Governor
**Date**: 2026-03-09

## Problem Statement

The repository has accumulated structural debt from earlier multi-environment proprietary build strategies. Now that FailSafe is fully open-source on GitHub, this separation provides no benefit and creates:

1. **Duplicate images** - 6 PNG files in root that duplicate `FailSafe/extension/media/`
2. **Redundant AI toolchain folders** - `FailSafe/Antigravity/`, `FailSafe/VSCode/`, `FailSafe/targets/` contain environment-specific skills/prompts that duplicate `.claude/` content
3. **Stale PROD-Extension folder** - Build artifacts and deployment packages that should be generated, not tracked
4. **Excessive README files** - 14+ README files with Socket.dev badges that drift out of sync

## Open Questions

None - this is a cleanup operation with clear scope.

## Phase 2: Remove Redundant AI Toolchain Folders

### Affected Files

- `FailSafe/Antigravity/` - DELETE entire folder (20 files)
- `FailSafe/VSCode/` - DELETE entire folder (20 files)
- `FailSafe/targets/` - DELETE entire folder (7 files)

### Rationale

| Folder | Purpose | Why Remove |
|--------|---------|------------|
| `Antigravity/` | Gemini/Antigravity skills | Duplicates `.claude/skills/` - Gemini can use same skills |
| `VSCode/` | Copilot prompts | Duplicates `.claude/skills/` - Copilot prompt format is outdated |
| `targets/` | Environment constraints | Build system no longer uses these constraints |

### Changes

```bash
git rm -r FailSafe/Antigravity/ FailSafe/VSCode/ FailSafe/targets/
```

### Note

The canonical skill location is `.claude/skills/` and `.claude/agents/`. All AI toolchains should reference these.
